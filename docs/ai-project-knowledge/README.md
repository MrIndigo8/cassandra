# CASSANDRA — база знаний для ИИ-анализа

Эта папка содержит **описание архитектуры, связей процессов, UI/UX и состояния качества** проекта CASSANDRA. Документы предназначены для автоматизированного или ручного анализа другими ИИ-агентами: их можно целиком или частично загружать в контекст.

## Как пользоваться

| Файл | Содержание |
|------|------------|
| [01-scope-and-methodology.md](./01-scope-and-methodology.md) | Что проверено, ограничения («каждая строка»), исключения из обзора |
| [02-architecture-layers.md](./02-architecture-layers.md) | Стек, слои, границы ответственности, единая система |
| [03-data-flow-and-pipeline.md](./03-data-flow-and-pipeline.md) | Потоки данных, таблицы, cron, связь подсистем |
| [04-api-inventory.md](./04-api-inventory.md) | Перечень HTTP API и роли в общей схеме |
| [05-database-schema-notes.md](./05-database-schema-notes.md) | Миграции, известные несоответствия схемы и кода |
| [06-ui-ux-inventory.md](./06-ui-ux-inventory.md) | Маршруты, layout, компоненты, паттерны UX, i18n |
| [07-qa-build-health-risks.md](./07-qa-build-health-risks.md) | Тесты, сборка, риски, технический долг, чек-лист QA |

Связанный документ в репозитории: [`../DATA_FLOW.md`](../DATA_FLOW.md) — краткий поток данных и Mermaid-диаграмма.

## Версия и актуальность

- Документы отражают состояние кодовой базы на момент генерации (анализ структуры `src/`, `supabase/migrations/`, `package.json`, прогон `npm run test` и `npm run build`).
- При крупных изменениях в продукте обновляйте соответствующие разделы вручную или регенерируйте описания по тому же шаблону.
