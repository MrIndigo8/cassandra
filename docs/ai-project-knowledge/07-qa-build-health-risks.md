## Охват тестами

- Покрыты **научные утилиты** (эмоции, социальный прогноз).
- **Нет** E2E и нет массового unit-покрытия API/React — регрессии в маршрутах и UI ловятся вручную или при расширении тестов.

## Риски работоспособности

### 1. Частота cron (Vercel Hobby)

- Один запуск в сутки для **`/api/cron`**.
- Touchpoints с задержками **2 ч / 24 ч / …** обрабатываются **`process-touchpoints`** только когда сработал cron; фактически уведомления могут приходить **позже запланированного интервала** (до ~24 ч).
- **Weekly digest** (`src/app/api/weekly-digest/route.ts`): при обычном вызове возвращает `skipped: true` с причиной `not_sunday`, если **день недели по UTC не воскресенье** (`getUTCDay() !== 0`). Параметр `?force=1` обходит проверку (для отладки). Ежедневный cron в 02:00 UTC сработает с генерацией дайджеста **только в воскресные дни UTC**.

### 2. Гонки анализа

- Одновременный вызов **`/api/analyze`** из cron и сразу после создания записи может обрабатывать одни и те же `entry_id`. Убедиться в идемпотентности обновлений `deep_analysis` и `entries`.

### 3. Схема БД

- См. [05-database-schema-notes.md](./05-database-schema-notes.md): `entries.type`, поля IP, имена таблиц в verification.

### 4. Локализация контента уведомлений

- Запланированные записи в `schedule-touchpoints` содержат **жёстко заданные русские** заголовки/сообщения в БД — для EN-пользователей UX неконсистентен.

### 5. NotificationBell и время

- Относительное время `timeAgoLabel` не локализовано (см. [06-ui-ux-inventory.md](./06-ui-ux-inventory.md)).

## Чек-лист ручного QA (критические сценарии)

1. Регистрация / вход → редирект на `/feed`.
2. Создание записи → запись в ленте, поля после анализа.
3. Колокольчик → панель не перекрывает шапку, закрытие по оверлею и Escape.
4. Админ: пользователь с ролью `observer` не открывает `/admin/*`.
5. Админ Psyche: загрузка данных с `/api/admin/psyche-data`, графики при непустых данных.
6. После деплоя: применение миграций **027**, **028** на Supabase.

## Переменные окружения (имена, без значений)

Типично требуются: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, серверные ключи Supabase для service role, `CRON_SECRET`, URL приложения (`NEXT_PUBLIC_APP_URL` / `VERCEL_URL`), ключ Anthropic. Точный список — по `process.env` в коде и документации деплоя.
